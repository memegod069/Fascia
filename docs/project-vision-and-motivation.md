# Fascia Project Vision & Motivation

This document captures the original intent and non-negotiable goals of the project.  
It was written based on the creator's own words and reasoning.

## The Core Equation

y = m · x

- **y** = The final high-quality result (Weta Tissue level or Red Dead Redemption 2 level creature soft-tissue)
- **x** = The user's own LLM / AI (the brain, the muscle TD)
- **m** = Fascia (the Blender addon / the "hands")

The goal is to figure out what the addon must provide so that a normal person + a good LLM can achieve professional-level results without needing expensive proprietary tools or hundreds of hours of manual work.

## Non-Negotiable Rule

**We will NOT compromise on quality or performance.**

- The target is real Weta / Red Dead Redemption level.
- Not "good enough".
- Not "looks plausible for an indie tool".
- Not a cheap approximation.
- No watering down the ambition.

If something only gets us to "close enough", we keep working until we can reach the real target.

## Why This Project Exists (Creator's Motivation)

The creator started because:
- Wanted to make high-quality creatures in Blender (example: a Red Dead Redemption level horse).
- Good base meshes can be generated with AI tools (Rodin, Meshy, etc.).
- But the soft tissue part (muscles, fascia, fat, skin that actually moves and deforms realistically) is extremely hard and time-consuming to do manually.
- Professional solutions (Weta, Ziva) are either internal, extremely expensive, or not available to normal people.
- Existing assets are costly or not accessible.
- Manual work in Blender requires hundreds of hours of focused effort.
- The creator saw old videos from big studios (around 10+ years ago) where they already had this technology and are still profiting from it.
- Goal: Reverse-engineer / open-source the process so normal people can do it too — like making "generic medication" instead of only having the expensive branded version.

The creator has no deep prior knowledge of Blender or advanced software development. The entire project was built by vibecoding with AI. This is intentional — the addon should be usable and steerable by LLMs.

## What Success Looks Like

When a person + a strong LLM uses Fascia, they should be able to create creature soft tissue that matches the visual quality and believable movement of top-tier games and VFX, without:
- Paying huge amounts for assets
- Spending hundreds of hours on manual sculpting and simulation setup
- Being locked behind proprietary tools

The project should feel like giving the LLM proper "hands" in Blender so it can actually deliver professional results.

## For Future AI Agents

Remember this every time you work on the project:

1. The user does **not** want the "cheap attar" version. They want the real thing.
2. Geometric approximations are temporary stepping stones only.
3. Every decision should be judged against whether it helps reach Weta / RDR level quality and performance.
4. The user is open-sourcing this because they want regular people to have access to this capability.
5. The creator is learning as they go. Keep explanations and architecture simple enough that an LLM + a non-expert can still drive progress.

Do not suggest lowering the bar. Push toward the real target.
